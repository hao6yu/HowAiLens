# HowAILens Service Ops

Quick commands for running the local backend and the XIAO-hosted frontend.

## What Runs Where

Backend:

- Runs on the Mac.
- Listens on `http://localhost:8787`.
- Receives JPEG uploads at `/analyze`.
- Calls OpenAI when `OPENAI_API_KEY` is set in `.env`.

Frontend:

- Runs on the XIAO ESP32S3 firmware.
- Hosts the device page at `http://<XIAO_IP>/`.
- Current known IP during development has been `http://192.168.50.47/`, but this can change after Wi-Fi reconnects.
- This is not a separate local frontend dev server. Starting or restarting it means uploading or resetting the board.

## Backend

Start in the foreground:

```sh
nvm use
node tools/local-backend/server.js
```

Stop foreground backend:

```text
Ctrl+C
```

Start in the background:

```sh
source ~/.nvm/nvm.sh
nvm use
nohup node tools/local-backend/server.js > /tmp/howailens-backend.log 2>&1 < /dev/null &
echo $! > /tmp/howailens-backend.pid
```

Check backend health:

```sh
curl http://127.0.0.1:8787/health
```

Watch backend logs:

```sh
tail -f /tmp/howailens-backend.log
```

Stop background backend:

```sh
kill "$(cat /tmp/howailens-backend.pid)"
rm -f /tmp/howailens-backend.pid
```

If the PID file is missing, stop whatever is listening on port `8787`:

```sh
lsof -tiTCP:8787 -sTCP:LISTEN | xargs kill
```

Restart background backend:

```sh
lsof -tiTCP:8787 -sTCP:LISTEN | xargs kill 2>/dev/null || true
source ~/.nvm/nvm.sh
nvm use
nohup node tools/local-backend/server.js > /tmp/howailens-backend.log 2>&1 < /dev/null &
echo $! > /tmp/howailens-backend.pid
curl http://127.0.0.1:8787/health
```

## XIAO Frontend/Firmware

Build firmware:

```sh
/Users/haoyu/.platformio/penv/bin/pio run
```

Upload firmware:

```sh
/Users/haoyu/.platformio/penv/bin/pio run -t upload
```

Open serial monitor:

```sh
/Users/haoyu/.platformio/penv/bin/pio device monitor -p /dev/cu.usbmodem101 -b 115200
```

If the USB port changes, list connected devices:

```sh
/Users/haoyu/.platformio/penv/bin/pio device list
```

Stop serial monitor:

```text
Ctrl+C
```

Restart the XIAO frontend:

```sh
/Users/haoyu/.platformio/penv/bin/pio run -t upload
```

Uploading resets the board and restarts the XIAO-hosted page. You can also press reset or unplug/replug USB-C.

Check the XIAO page:

```sh
curl http://192.168.50.47/
```

If the IP changes, read it from serial output after boot:

```text
Open this URL: http://<XIAO_IP>
```

## Manual Test Flow

Use this when testing the full loop:

```sh
# Terminal 1: backend
nvm use
node tools/local-backend/server.js
```

```sh
# Terminal 2: firmware upload and monitor
/Users/haoyu/.platformio/penv/bin/pio run -t upload
/Users/haoyu/.platformio/penv/bin/pio device monitor -p /dev/cu.usbmodem101 -b 115200
```

Then tap the touch sensor. Expected flow:

```text
touch tap
-> camera captures JPEG
-> XIAO POSTs JPEG to backend
-> backend returns compact JSON text
-> OLED displays the text
-> XIAO / page shows last capture, upload status, and last AI text
```

Useful URLs:

```text
http://127.0.0.1:8787/
http://127.0.0.1:8787/health
http://127.0.0.1:8787/last-analysis
http://192.168.50.47/
http://192.168.50.47/last
```

## VS Code

PlatformIO is the best way to build, upload, and monitor the XIAO firmware from VS Code:

- Build: PlatformIO checkmark
- Upload: PlatformIO right-arrow
- Monitor: PlatformIO plug icon

The existing `.vscode/launch.json` is generated by PlatformIO and is ignored by git. It is mainly for firmware debugging with a supported debug setup, not for starting the normal upload/monitor workflow.

You can use VS Code `launch.json` for the Node backend. Add a local configuration like this if you want a Run and Debug button:

```json
{
  "type": "node",
  "request": "launch",
  "name": "HowAILens Backend",
  "program": "${workspaceFolder}/tools/local-backend/server.js",
  "cwd": "${workspaceFolder}",
  "runtimeExecutable": "node",
  "console": "integratedTerminal"
}
```

If VS Code does not pick up the Node version from `nvm`, start VS Code from a terminal after running `nvm use`, or keep using the terminal command:

```sh
nvm use
node tools/local-backend/server.js
```
